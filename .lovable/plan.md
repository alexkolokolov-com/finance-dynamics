## Что не так с текущей листалкой

`react-pageflip` сейчас не работает по нескольким причинам, и они складываются:

1. **Дети без `forwardRef`.** Библиотека через `cloneElement` навешивает ref на каждый дочерний элемент. Наш `<Page>` — обычная функция-компонент без `forwardRef`, поэтому страницы либо не монтируются, либо рендерятся «мёртвыми» и листание молча не происходит.
2. **Конфликт пропсов размера.** Одновременно заданы `size="stretch"`, `autoSize`, `width/height`, `minWidth/maxWidth`, `usePortrait=true`. В таком сочетании контейнер часто схлопывается в 0×0 или фиксируется в одностраничном режиме, и клики/свайпы не доходят.
3. **`usePortrait={true}` запрещает двухстраничный разворот** — даже на десктопе будет одна страница, что противоречит исходному запросу.
4. **Кнопки ←/→ дёргают `bookRef.current.pageFlip()` без проверки готовности** — на первый клик после загрузки часто `undefined`.
5. Сам `react-pageflip` давно не поддерживается, ломается со StrictMode React 18 и регулярно «не отдаёт» pageFlip API.

## План починки

Идём по пути замены неработающего пакета на самодельный двухстраничный flip — он надёжнее, легче и контролируемее, и сохраняет визуал «разворот книги с перелистыванием».

### Шаги

1. **Диагностика (короткая).** Запустить Playwright на `/cashback#preview`, проверить, что контейнер листалки имеет ненулевой размер и что кнопки реально вызывают смену страницы. Снять скриншоты до/после клика. Если выяснится, что проблема — только в `forwardRef` + пропсах, оставим `react-pageflip` (вариант B). Базово идём вариантом A.

2. **Вариант A — custom CSS flip (рекомендуемый).**
   - Удалить зависимость `react-pageflip` из `CashbackPreviewFlip.tsx` (пакет оставим в `package.json`, чтобы не трогать lock без нужды; можно вычистить позже).
   - Переписать компонент: контейнер с `perspective: 2000px`, внутри ряд из двух «слотов» (левая и правая страница текущего разворота). Сами страницы — абсолютно позиционированные `<div>` с `transform-style: preserve-3d` и `transform: rotateY(...)`, переключаемые по индексу. Эффект — плавный поворот правой страницы на −180° (вперёд) и левой на +180° (назад), 600 мс ease-in-out, с лёгкой тенью на «обороте».
   - Состояние: `spread` (индекс разворота 0…⌈N/2⌉−1), `flipping` (`'next' | 'prev' | null`). По окончании transition увеличиваем/уменьшаем `spread`, сбрасываем `flipping`.
   - Двухстраничный режим только при `min-width: 768px`; на мобиле — одна страница, кнопки листают по одной (сейчас и так так выглядит лучше).
   - Управление: кнопки ←/→, клавиши ←/→ (`useEffect` + `keydown`), свайп (touchstart/touchend, порог 40 px). Колесо мыши не трогаем.
   - Индикатор «страница X–Y из 9» оставляем, считаем из `spread`.
   - Сохраняем текущие тёмный фон секции, заголовок «Полистайте первые страницы» и подпись снизу.

3. **Вариант B — fallback, если поведение реально нужно от `react-pageflip`.**
   - Обернуть `Page` в `React.forwardRef<HTMLDivElement, PageProps>` и пробросить `ref` на корневой `div`.
   - Убрать `autoSize`, `usePortrait`, `size="stretch"`, оставить только `width/height` + `minWidth/maxWidth/minHeight/maxHeight`. Включить двухстраничный режим (`usePortrait={false}`).
   - Делать клики через `bookRef.current?.pageFlip?.()?.flipNext?.()` (опциональные вызовы) и инициализировать только после `onInit`.

4. **Проверка (Playwright, headless).**
   - Открыть `http://localhost:8080/cashback`, проскроллить к `#preview`.
   - Скриншот: видны две страницы рядом (desktop 1280 px) / одна (mobile 390 px).
   - Кликнуть «→», подождать 700 мс, скриншот: показан следующий разворот, счётчик обновился до «3–4 из 9».
   - Кликнуть «←», вернуться к «1–2 из 9», скриншот.
   - На мобильном viewport (`page.set_viewport_size({width: 390, height: 800})`) проверить, что показывается одна страница и кнопки листают по одной.
   - Проверить отсутствие ошибок в `page.on("pageerror")` и `console`.

### Технические детали (вариант A)

```text
spread 0: [1 | 2]   spread 1: [3 | 4]  ...  spread 4: [9 | —]
```

- Контейнер: `width: min(960px, 100%)`, `aspect-ratio: 2 / 1.414` (двухстр.) / `1 / 1.414` (одностр.).
- Каждая страница — `position:absolute; inset:0 50% 0 0` (левая) / `inset:0 0 0 50%` (правая); при перелистывании клонируется поверх «уходящая» страница с `transform-origin: left center` или `right center` и анимируется `rotateY`.
- Тень оборота: `linear-gradient` поверх с `opacity` от 0 до 0.35 синхронно с поворотом.
- Все цвета/тени — через токены (`bg-background`, `shadow-lg` — оставляем существующие).
